# Real-Time Particle Event Trigger System Simulation

## 1. Problem Statement
In high-energy physics experiments such as the Large Hadron Collider (LHC) at CERN, millions of particle collisions occur every second. The data generated by these collisions is immense, far exceeding the storage bandwidth of modern computing systems. To make breakthroughs, physicists rely on **Trigger Systems**—real-time processing pipelines that make split-second decisions to discard uninteresting background "noise" events while retaining rare and theoretically valuable "signal" events. Developing efficient triggers is one of the most complex challenges in modern data acquisition.

## 2. System Architecture
This project implements an end-to-end simulation of a simplified particle trigger pipeline.
The data flow is as follows:
**Event Generation → Filtering (Trigger) → Evaluation (Metrics/Visualization)**

1.  **Event Generation**: A synthetic particle event stream is generated containing kinematic variables like `energy`, `momentum`, and detector `noise_level`. True signals (hidden ground truth) are mixed into an overwhelming sea of background events based on a configurable Signal-to-Noise Ratio (SNR).
2.  **Trigger System**: The incoming data stream is evaluated. A binary decision (Accept/Reject) is made for every single event.
3.  **Metrics Evaluation**: The system's performance is graded on physics metrics: Signal Efficiency (Recall) and Background Rejection Rate (True Negative Rate), alongside Precision and F1 Score.

## 3. Trigger Methods

This repository offers three implemented strategies for filtering data:

*   **Rule-Based Logic**: The most basic trigger approach (often used as Level-1 triggers in hardware). It applies strict mathematical inequalities, for example, accepting an event only if $Energy > \text{Threshold}$ and $Noise < \text{Threshold}$. It is extremely fast but lacks nuance and generalization.
*   **Statistical Approach**: Models the background noise data distributions (e.g., probability density mappings). It utilizes hypothesis testing, triggering an "accept" when an event constitutes an anomaly (high significance / low p-value) against the expected background model.
*   **Machine Learning (ML) Approach**: A multi-dimensional feature analysis. By looking at all sensory inputs simultaneously, algorithms like Logistic Regression or Random Forests can learn hidden correlations between variables to draw highly efficient, nonlinear decision boundaries.

## 4. Results

*You can generate and view real-time Results in the Streamlit App or the Jupyter Notebook.*

When comparing the triggering systems:
*   **Rule-Based** is prone to throwing away legitimate edge cases or accepting artificial noise spikes. It relies heavily on arbitrary threshold guesswork.
*   **Statistical** cleanly eliminates uniform backgrounds but fails to accurately identify signals when the underlying backgrounds are heavily non-Gaussian or highly skewed.
*   **Machine Learning** generally offers the highest Signal Efficiency and F1 scores by adapting to the exact joint feature distributions. However, deploying ML inference hardware directly on sensor readouts can be computationally complex in actual research environments.

## 5. Real-World Relevance

This repository serves as an educational and procedural proxy for real-world physics data acquisition:
*   **Particle Detectors**: Approximates the Level-1 (hardware) and High-Level Trigger (software farm) structures used by CMS/ATLAS.
*   **Real-Time Data Filtering**: High throughput analysis applies beyond particle physics, moving into automated stock trading algorithms and astrophysics transient phenomenon detection.
*   **Data Pipelines**: Highlights the importance of statistical evaluations before expensive data is persisted to disk storage.

## Getting Started

### Installation
```bash
python -m venv .venv
source .venv/bin/activate  # Or .venv\Scripts\Activate.ps1 on Windows
pip install -r requirements.txt
```

### Running the App
Launch the interactive dashboard to simulate streams and configure thresholds:
```bash
streamlit run app.py
```

Check out `notebooks/trigger_analysis.ipynb` for an exploratory data analysis walkthrough of the methods.
